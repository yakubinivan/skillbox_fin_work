# Финальная практическая работа:

## 1. Краткое описание проекта
Проект предназначен для развертывания защищенной корпоративной сети в Yandex Cloud. Включает в себя автоматизированный Удостоверяющий центр (CA), VPN-шлюз для сотрудников, систему мониторинга состояния серверов и автономную систему бэкапов.
**Внимание:** Перед запуском инфраструктуры обязательно замените публичный SSH-ключ в файлах директории deploy/ на ваш собственный.

## 2. Требования к инфраструктуре и инструменты
Для развертывания проекта вам потребуются:
* **Аккаунт в Yandex Cloud** с настроенным платежным аккаунтом.
* **yc CLI** — интерфейс командной строки Yandex Cloud.
* **Docker & Docker Compose** — для запуска мониторинга.
* **SSH-ключ** — необходим для доступа к узлам (замените на свой в файлах директории deploy/).

## 4. Сетевая безопасность (Firewall)
Защита периметра реализована через **Security Groups** Yandex Cloud. Скрипт `setup_infrastructure.sh` автоматически настраивает следующие правила:
* **Входящий трафик (Ingress):** Разрешены порты 22 (SSH), 1194 (OpenVPN) и 3000 (Grafana). Остальные входящие соединения из внешней сети заблокированы.
* **Внутренняя сеть:** Разрешен полный обмен трафиком между серверами внутри подсети для работы системы мониторинга и передачи бэкапов в репозиторий Borg.

## 5. Работа с PKI и VPN-сервером
### Жизненный цикл PKI:
1. **Инициализация:** На `ca-server` выполняется `init-pki` и создается корневой сертификат (`build-ca`).
2. **Выпуск сертификатов:** `vpn-server` генерирует запрос на сертификат (CSR), который подписывается на CA. Секретный ключ CA хранится только на `ca-server`.
3. **Безопасность:** Для аутентификации используется связка «сертификат + TLS-auth», что защищает VPN-сервер от DoS-атак и сканирования портов.

### Конфигурация VPN:
Сервер функционирует как шлюз (gateway) между внешними клиентами и облачной инфраструктурой:
* **Протокол и порт:** UDP 1194 (обеспечивает высокую скорость и низкие задержки).
* **Туннель:** Используется виртуальный интерфейс `tun`, клиенты получают IP из сети `10.8.0.0/24`.
* **Маршрутизация:** Через механизм `push "route 192.168.10.0 255.255.255.0"` клиентам автоматически передаются правила доступа к внутренней сети проекта.
* **Автоматизация:** Конфигурация подтягивается из файла `vpn-config/server.conf` и применяется автоматически при развертывании.

## 6. Описание файлов и их назначение
* `setup_infrastructure.sh` — Автоматизация создания ресурсов в Yandex Cloud (сети, подсети, ВМ).
* `deploy/` — Директория с ролевыми конфигурациями Cloud-Init (CA.yaml, vpn.yaml, monitoring.yaml). Позволяет избежать избыточной установки ПО. **Требует замены SSH-ключа на пользовательский в секции ssh-authorized-keys.**
* `vpn-config/server.conf` — Файл конфигурации VPN-сервера.
* `/package` — Исходный код кастомного .deb пакета для автоматизации конфигурации Easy-RSA.
* `/monitoring` — Конфигурация стека мониторинга (Prometheus + Grafana).
* `/scripts/backup.sh` — Скрипт автоматизации бэкапа в репозиторий Borg.

## 7. Как развернуть проект
1. Склонируйте репозиторий.
2. Запустите `./setup_infrastructure.sh` для создания облачных ресурсов.
3. После готовности ВМ установите пакет из `/package` на **ca-server**
4. На сервере мониторинга перейдите в папку `/monitoring` и запустите `docker-compose up -d`. Для визуализации используйте дашборд Node Exporter (**ID: 1860**).

## 8. Работа с VPN-клиентами
Для создания доступа нового клиента на **ca-server**:
1. Выполните генерацию: `./easyrsa build-client-full client_name`.
2. Полученные файлы `ca.crt`, `client_name.crt` и `client_name.key` передайте пользователю для настройки подключения.

## 9. План развития инфраструктуры (Roadmap)
* Переход на **Terraform** для декларативного управления IaC.
* Использование **Ansible** для централизованного управления конфигурациями.
* Внедрение **HashiCorp Vault** для безопасного хранения секретов и ключей.

## 10. Дополнительные комментарии
