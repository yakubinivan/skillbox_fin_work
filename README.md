# Финальная практическая работа:

## 1. Краткое описание проекта
Проект предназначен для развертывания защищенной корпоративной сети в Yandex Cloud. Включает в себя автоматизированный Удостоверяющий центр (CA), VPN-шлюз для сотрудников, систему мониторинга состояния серверов и автономную систему бэкапов.

## 2. Требования к инфраструктуре и инструменты
Для развертывания проекта вам потребуются:
* **Аккаунт в Yandex Cloud** с настроенным платежным аккаунтом.
* **yc CLI** — интерфейс командной строки Yandex Cloud.
* **Docker & Docker Compose** — для запуска мониторинга.
* **SSH-ключ** — для доступа к узлам инфраструктуры.

## 3. Состав серверов

| Имя сервера | Роль | Внешний IP |
|:--- |:--- |:--- |
| **ca-server** | Удостоверяющий центр (Easy-RSA) | 89.169.137.5 |
| **vpn-server** | Шлюз доступа (OpenVPN) | 111.88.246.94 |
| **monitoring-server** | Метрики (Prometheus+Grafana) и Borg Repo | 62.84.125.193 |

## 4. Схема взаимодействия
1. Администратор разворачивает ВМ через скрипт `setup_infrastructure.sh`.
2. На **ca-server** устанавливается пакет `.deb`, настраивающий параметры выпуска сертификатов.
3. **vpn-server** использует сертификаты от CA для аутентификации клиентов.
4. **monitoring-server** собирает метрики со всех узлов и хранит инкрементальные бэкапы директории `/my-pki`.

## 5. Описание файлов и их назначение
* `setup_infrastructure.sh` — Автоматизация создания ресурсов в Yandex Cloud (сети, подсети, ВМ).
* `cloud-config.yaml` — Инструкции по инициализации пользователей и безопасности при создании ВМ.
* `/package` — Исходный код кастомного .deb пакета для автоматизации конфигурации Easy-RSA.
* `/monitoring/docker-compose.yml` — Описание стека мониторинга (контейнеризация).
* `/monitoring/prometheus.yml` — Конфигурация целей (targets) для сбора метрик.

## 6. Как развернуть проект
1. Склонируйте репозиторий.
2. Запустите `./setup_infrastructure.sh` для создания облачных ресурсов.
3. После готовности ВМ установите пакет из `/package` на CA-сервер.
4. На сервере мониторинга запустите `docker-compose up -d`.

## 7. План развития инфраструктуры (Roadmap)
* Переход на **Terraform** для декларативного управления IaC.
* Использование **Ansible** для централизованного управления конфигурациями.
* Внедрение **HashiCorp Vault** для безопасного хранения секретов и ключей.

## 8. Дополнительные комментарии
Все бэкапы в системе BorgBackup выполняются по расписанию и используют авторизацию по ключам, что исключает необходимость ручного ввода паролей.
