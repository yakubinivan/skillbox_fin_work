# Финальная практическая работа:

## 1. Краткое описание проекта
Проект предназначен для развертывания защищенной корпоративной сети в Yandex Cloud. Включает в себя автоматизированный Удостоверяющий центр (CA), VPN-шлюз для сотрудников, систему мониторинга состояния серверов и автономную систему бэкапов.
**Внимание:** Перед запуском инфраструктуры обязательно замените публичный SSH-ключ в файлах директории `deploy/` (`CA.yaml`, `vpn.yaml`, `monitoring.yaml`) в секции `ssh-authorized-keys` на ваш собственный.

## 2. Требования к инфраструктуре и инструменты
Для развертывания проекта вам потребуются:
* **Аккаунт в Yandex Cloud** с настроенным платежным аккаунтом.
* **yc CLI** — интерфейс командной строки Yandex Cloud, авторизованный в вашем облаке.
* **Docker & Docker Compose** — предустанавливаются автоматически на сервер мониторинга через Cloud-init (требуется только запуск контейнеров).
* **SSH-ключ** — публичная часть ключа должна быть прописана в конфигурационных файлах перед созданием ВМ.

## 4. Сетевая безопасность (Firewall)
Защита периметра реализована через **Security Groups** Yandex Cloud. Скрипт автоматизации настраивает следующие правила:
* **Входящий трафик (Ingress):** 
    * Порт `22/TCP` (SSH) — для управления серверами.
    * Порт `1194/UDP` (OpenVPN) — доступ клиентов к VPN.
    * Порт `3000/TCP` (Grafana) — веб-интерфейс мониторинга.
* **Внутренняя сеть (192.168.10.0/24):** Разрешен полный обмен трафиком между серверами для сбора метрик (Node Exporter) и передачи бэкапов в репозиторий Borg. Все входящие соединения из внешней сети, не описанные выше, заблокированы.

## 5. Работа с PKI и VPN-сервером
### Жизненный цикл PKI:
1. **Инициализация:** На `ca-server` после установки пакета выполняется инициализация структуры `./easyrsa init-pki` и создание корневого сертификата `build-ca`.
2. **Выпуск сертификатов:** 
   * Генерируется запрос на сертификат (CSR) для сервера.
   * Запрос подписывается на CA, формируя файл сертификата сервера.
   * Секретный ключ CA (`ca.key`) никогда не покидает `ca-server`, что гарантирует безопасность инфраструктуры.
3. **Безопасность:** Для аутентификации используется связка «сертификат + TLS-auth (ta.key)». Это обеспечивает проверку подлинности клиента на уровне TLS, защищая OpenVPN от DoS-атак и сканирования портов (сервер не отвечает на пакеты без валидного HMAC-ключа).

### Конфигурация VPN:
Сервер функционирует как шлюз (gateway) между внешними клиентами и облачной инфраструктурой:
* **Протокол и порт:** UDP 1194 (обеспечивает высокую скорость и низкие задержки).
* **Туннель:** Используется виртуальный интерфейс `tun`, клиенты получают IP из сети `10.8.0.0/24`.
* **Маршрутизация:** Через механизм `push "route 192.168.10.0 255.255.255.0"` клиентам автоматически передаются правила доступа к внутренней сети проекта.
* **Forwarding:** Для корректной работы шлюза на сервере автоматически активируется `net.ipv4.ip_forward = 1` через Cloud-init.
* **Автоматизация:** Конфигурация подтягивается из файла `vpn-config/server.conf` и применяется автоматически при развертывании.

## 6. Описание файлов и их назначение
* `setup_infrastructure.sh` — Автоматизация создания ресурсов в Yandex Cloud (сети, подсети, ВМ).
* `deploy/` — Директория с ролевыми конфигурациями Cloud-Init (CA.yaml, vpn.yaml, monitoring.yaml). Позволяет избежать избыточной установки ПО. **Требует замены SSH-ключа на пользовательский в секции ssh-authorized-keys.**
* `vpn-config/server.conf` — Файл конфигурации VPN-сервера.
* `/package` — Исходный код кастомного .deb пакета для автоматизации конфигурации Easy-RSA.
* `/monitoring` — Конфигурационные файлы для стека мониторинга (файл `docker-compose.yaml`, настройки Prometheus и Grafana).
* `/scripts/backup.sh` — Скрипт автоматизации бэкапа в репозиторий Borg.

## 7. Как развернуть проект
1. **Подготовка:** Склонируйте репозиторий и замените SSH-ключ в файлах директории `deploy/` на ваш собственный.
2. **Инфраструктура:** Запустите `./setup_infrastructure.sh` для автоматического создания облачных ресурсов.
3. **Настройка PKI:** Зайдите на **ca-server**, установите пакет и инициализируйте Удостоверяющий Центр:
   ```bash
   sudo dpkg -i /package/custom-easy-rsa.deb
   cd /etc/easy-rsa/
   sudo ./easyrsa init-pki
   sudo ./easyrsa build-ca nopass
   ```
4. **Запуск VPN:** На **ca-server** выпустите сертификат для сервера и перенесите его вместе с `ca.crt` на **vpn-server** в директорию `/etc/openvpn/server/`. После чего запустите службу:
   ```bash
   sudo systemctl enable --now openvpn@server
   ```
5. **Мониторинг:** На **monitoring-server** перейдите в папку `/monitoring` и запустите стек:
   ```bash
   sudo docker-compose up -d
   ```
   Для визуализации метрик используйте Grafana (порт 3000) и импортируйте дашборд Node Exporter (**ID: 1860**).

## 8. Работа с VPN-клиентами
Для создания доступа нового клиента на **ca-server**:
1. **Генерация:** Выполните команду для создания ключей клиента:
   ```bash
   sudo ./easyrsa build-client-full client_name nopass
   ```
2. **Передача файлов:** Передайте пользователю следующий набор файлов для настройки подключения:
   * `ca.crt` — корневой сертификат.
   * `client_name.crt` — сертификат клиента.
   * `client_name.key` — закрытый ключ клиента.
   * `ta.key` — ключ TLS-auth (необходим для защиты канала управления).

## 10. Дополнительные комментарии
* **Автоматизация бэкапов:** Все резервные копии в системе **BorgBackup** выполняются по расписанию. Для активации ежедневного задания в 02:00 необходимо добавить скрипт в планировщик:
  ```bash
  (crontab -l ; echo "0 2 * * * /scripts/backup.sh") | crontab -
  ```
* **Безопасность:** Доступ к репозиторию бэкапов осуществляется по SSH-ключам, что исключает необходимость ручного ввода паролей и позволяет системе работать полностью автономно.
