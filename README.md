# skillbox_fin_work

## 1. Описание проекта
Данный проект автоматизирует развертывание защищенной сетевой инфраструктуры в Yandex Cloud. Основные компоненты:
* **Удостоверяющий центр (CA)** для выпуска сертификатов.
* **VPN-шлюз** для защищенного доступа.
* **Система мониторинга** на базе стека Prometheus + Grafana.
* **Система резервного копирования** через BorgBackup.

## 3. Требования к инфраструктуре
* Активный аккаунт в Yandex Cloud.
* Установленный и настроенный `yc CLI`.
* Наличие SSH-ключа для доступа к ВМ.

## 4. Инструменты автоматизации
1. **IaC (Infrastructure as Code):** Скрипты развертывания через `yc CLI`.
2. **Конфигурация:** Кастомный `.deb` пакет `skillbox-ca-config` для автоматической настройки Easy-RSA на CA-сервере.
3. **Оркестрация:** Развертывание мониторинга через Docker Compose.

## 5. Как развернуть проект
1. **Развертывание ВМ:** Выполните скрипт `setup_infrastructure.sh`. Перед запуском убедитесь, что файл `cloud-config.yaml` находится в той же директории.
2. **Настройка CA:** Перейдите в папку `/package` и установите подготовленный пакет. Это автоматически настроит переменные для выпуска сертификатов (файл `vars`).
3. **Запуск мониторинга:** На сервере мониторинга перейдите в директорию `/monitoring` и выполните команду `docker-compose up -d`.

## 6. Описание файлов репозитория
* `setup_infrastructure.sh` — Bash-скрипт создания ВМ в облаке.
* `cloud-config.yaml` — Конфигурация для автоматического создания пользователя и добавления SSH-ключей.
* `/package` — Исходный код и структура кастомного .deb пакета для автоматизации PKI.
* `/monitoring` — Конфигурационные файлы для Docker Compose и Prometheus.

## 7. Схема взаимодействия
Клиент подключается к **VPN-server** по защищенному каналу. Аутентификация происходит на основе сертификатов, выпущенных на **CA-server**. Состояние всех узлов отслеживается **monitoring-server**, на который также ежедневно копируются бэкапы ключей через BorgBackup.
